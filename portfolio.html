The highlights

Quiz World

Technologies used:

  • Angular
  • ASP.NET Core
  • MS SQL Server

Quiz World is an app that allows users to create and take quizzes, featuring single-choice, multiple-choice, and text-based questions. The user interface, built with Angular Material, offers an excellent user experience for quizzes and quiz creation. It supports sorting, pagination, and role-based authorization. Quiz World leverages ASP.NET Core's Web API to a full extent and utilizes a Redis database for securely managing user logouts. The front-end and back-end projects include over 500 unit, integration, and component tests. The database architecture is driven by Entity Framework Core.

Vue Flashcards

Technologies used:

  • Vue
  • TypeScript
  • NestJS
  • PostgreSQL

Originally, I had a flashcards project written in React and Express (with TypeScript). However, because it was one of my first serious projects, both the client and the server's codebases ended up quite messy. To revive the concept, I decided to redo the whole thing using NestJS and Vue to practice these frameworks. The new version has a cleaner look, thanks to the Vuetify component library. Vue Flashcards lets you make flashcards for effective studying. Once you create a deck, everyone can use it and bookmark it. Additionally, users can personalize their color preferences for a more customized experience. Both the client and the server have numerous tests; the Vue application has mostly component and end-to-end tests with a few unit tests, wheres the NestJS application has many unit, integration, and end-to-end tests.

React Ecommerce

Technologies used:

  • React
  • TypeScript
  • Express
  • MongoDB

React Ecommerce is a rather weird store, not only because you don't use real money to purchase things, but also because you cannot buy the same product multpile times. Anyways, this project utilizes TypeScript on both the front end and the back end, with Mongoose powering the MongoDB database. The user interface is achieved with the help of Material UI. The user is capable of searching, sorting, and paginating lists of products. Logged in users can adjust color preferences and track the products which they have "bought". The projects together feature 200+ tests, most of which are end-to-end ones. Front-end tests were written in Playwright and back-end tests were written in Mocha and Chai.
