The highlights

Cardflow

Technologies used:

  • React
  • TypeScript

Cardflow is a web application that allows users to buy, sell, and trade TCG cards (currently supporting Yu-Gi-Oh cards only), which I develop in collaboration with other developers. I am the main front-end enginner on the team and am responsible for almost all crucial aspects and features of the front end. Some of those include making the web pages responsive, developing various client-side APIs for concerns like authentication and the shopping cart, implementing internalization, and creating web pages in accordance to provided Figma files.

Some minor tasks I have been responsible for include implementing a few bug fixes on our Django server, reviewing pull requests from external contributors, and submitting detailed bug reports when I encountered a problem with the back end.

Quiz World

Technologies used:

  • Angular
  • ASP.NET Core
  • MS SQL Server

Quiz World is an app that allows users to create and take quizzes, featuring single-choice, multiple-choice, and text-based questions. The user interface, built with Angular Material, offers an excellent user experience for quizzes and quiz creation. It supports sorting, pagination, and role-based authorization. Quiz World leverages ASP.NET Core's Web API to a full extent and utilizes a Redis database for securely managing user logouts. The front-end and back-end projects include over 500 unit, integration, and component tests. The database architecture is driven by Entity Framework Core.

Vue Flashcards

Technologies used:

  • Vue
  • TypeScript
  • NestJS
  • PostgreSQL

Originally, I had a flashcards project written in React and Express (in TypeScript). However, because it was one of my first serious projects, both the client and the server's codebases ended up quite messy. To revive the concept, I decided to redo the whole thing using NestJS and Vue to practice these frameworks. The new version has a cleaner look, thanks to the Vuetify component library. Vue Flashcards lets you make flashcards for effective studying. Once you create a deck, everyone can use it and bookmark it. Additionally, users can personalize their color preferences for a more customized experience. Both the client and the server have numerous tests; the Vue application has mostly component and end-to-end tests with a few unit tests, wheres the NestJS application has many unit, integration, and end-to-end tests.
